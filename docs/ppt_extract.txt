Introduction 
3

Career platforms often give generic advice without clear job-fit reasoning.
PathFinder AI is a web-based career guidance system for students and job seekers.
The system supports two roles: job seeker and recruiter.
It analyzes profiles and job descriptions to show skill gaps, learning roadmaps, and ranked job matches.
Core innovation: importance-aware weighted job-fit scoring with explainable results.
Supporting technologies: FastAPI, React, SQLite, scikit-learn, sentence-transformers, and Google Gemini.


--- SLIDE 4 ---
Problem statement:
Traditional systems rely on simple similarity and lack explainability.
User skills are not aligned clearly with real job requirements.

--- SLIDE 5 ---
PathFinder AI connects user profiles with job requirements through a structured ML pipeline.
Skills are extracted using a taxonomy (skills.json) and fuzzy matching (rapidfuzz).
Each job skill receives an importance weight from the job description context.
Fit score combines matched skills, missing-skill penalties, experience, and a small bonus.
The system returns scores, matched skills, missing skills, and text explanations.
Gemini generates roadmaps, skill-gap analysis, and chat-based career guidance.
A lightweight contextual bandit adapts roadmap task recommendations from user feedback.
Proposed Solution Overview 
5

--- SLIDE 6 ---
Key Idea / Innovation:
Combines LLMs + Retrieval-Augmented Generation (RAG) for context-aware recommendations.
Uses skill gap analysis to generate personalized learning roadmaps.
Uses Reinforcement Learning-based adaptive logic to refine recommendations based on user feedback.
Introduces importance-aware Weighted Job-Fit Scoring for explainable and realistic recommendations.
System Workflow:
User or recruiter registers and logs in.
Seeker completes profile and optionally uploads a resume.
System extracts skills from profile and job descriptions.
Skill-gap and fit scoring compare user vs job.
System recommends careers (KNN) and ranks jobs (weighted fit).
Gemini generates a personalized roadmap for a selected job.
User saves roadmaps and continues learning.
Bandit logic updates roadmap actions based on interaction feedback.

--- SLIDE 15 ---
Comparison with SOTA Methods:
Uses weighted job-fit scoring, Gemini LLM, and KNN for a complete guidance flow
More explainable than cosine similarity or bag-of-skills matching alone
More job-specific than static rule-based career suggestions
Strengths:
Modular FastAPI architecture with separate prototype and product APIs
End-to-end flow: profile → gap analysis → roadmap → job ranking
Dual roles: job seeker and recruiter on one platform
Clear API design with documented endpoints (Swagger)
Limitations:
Skill extraction limited by skills.json taxonomy
Fit formula uses fixed weights; not trained on hiring outcomes
RAG requires optional ChromaDB setup
Contextual bandit is lightweight, not full deep reinforcement learning
Jobs are stored locally; external job feeds are not integrated yet

--- SLIDE 16 ---
Challenges:
Accurate skill extraction from resumes and free-text profiles
Unstructured and varying job description formats
Keeping career recommendation scores meaningful and non-zero
Integrating multiple AI components (ML pipeline, Gemini, optional RAG)
Managing two backend services (ports 8001 and 8002)
Response time on first ML model load

Solutions:
Skill taxonomy and rapidfuzz for normalization and matching
JD parsing, synthetic JD from skill lists, and semantic weighting (MiniLM)
Map user skills to KNN model vocabulary before encoding
Modular services: extractor, weight engine, fit scorer, Gemini, optional RAG/bandit
Clear separation: main app (auth, jobs, roadmaps) vs prototype (CSV scoring)
Model caching and documented warm-up for demonstrations
